# 版本亮点 — 1.1.0（2026-01-05）

## 重点功能

1. 👹 **质疑官（Devil's Advocate）—— 闭环质疑机制**
   - 质疑官在议题拆解阶段以及每轮总结后，输出结构化的质疑清单与盲点描述，并按严重度（Critical / Warning / Minor）分类。
   - 议长（Leader）必须在下一轮摘要中回应质疑，若存在关键问题，最终轮将触发强制修订。

2. 🔄 **用户参与式报告修订与版本管理**
   - 报告底部新增“💬 修订反馈”浮动面板，用户可直接提交修改建议或要求。
   - 系统首次修订前会备份原始报告为 `report_v0.html`，后续修订保存为 `report_v1.html`、`report_v2.html` 等；报告界面提供版本选择下拉框以便对比与回退。
   - 新增 `Report Auditor` 智能体负责分析用户反馈并生成修订草案，用户可在线预览并一键应用修订。

## 使用说明（快速）

1. 生成报告后，打开报告页面底部的“💬 修订反馈”面板并填写修改意图。
2. 等待系统生成修订建议，预览修改摘要与变更清单。
3. 点击“应用修订”将修改写入主报告（系统会自动创建修订版本）。

## 兼容性与注意事项

- 原始报告将被保留为 `report_v0.html`，请在重要场景中保留备份以便追溯。
- 若您使用本地 Ollama，请确保本地服务可用，否则系统会回退到配置的其他后端。
